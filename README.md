# Hype Pilot

## Project Description
Hype Pilot is a task management web application designed to help users organize their work, track deadlines, and stay
productive. It provides a simple interface for creating tasks, assigning priorities, and visualizing deadlines in a
calendar view.

## Features
- Create and manage tasks with due dates
- View deadlines in a calendar layout
- Priority labels (Top Priority / Mid Priority / Low Priority)
- Track task completion progress
- Receive reminder alerts
- Persistent storage using local storage

## Week 12 Excercise: Authentication feature
- This project includes user authentication using Supabase.
- Users can create accounts with an email and password
- Email confirmation is required for account activation
- Each user has access only to their own tasks
- Supabase is used for both authentication and database management

## Tech Stack
- React
- JavaScript
- CSS
- LocalStorage
- Supabase (authentication + database)

## Deployment
The application will be deployed using Vercel.

## Set-up instructions
- Phase 1 → useTasks hook + localStorage
- Phase 2 → TaskForm → TaskCard → TaskList → wire up in App.js
- Phase 3 → FilterBar + priority display
- Phase 4 → CalendarView + dateHelpers
- Phase 5 → ReminderBanner
- Build and test each phase before moving to the next. This keeps the app   functional at every step and makes bugs easy
to isolate.

## bugs or limitations
- UUID issue in testing crypto.randomUUID() is not supported in Jest/jsdom
Fix: using a custom ID generator

- App test query conflict: getByText matched multiple elements. 
Fix: using more specific selectors like getByRole

- CalendarView heading test failure: Fix: Updated tests to target .calendar-grid

- App heading test failure: Fix: Updated test to use getByText and verify navigation buttons.

- TaskCard date assertion failure: Fix: using regex-based assertions

- Dead code in TaskCard.js: PRIORITY_CLASS (lines 1–5) duplicates CARD_BORDER and is unused.
Fix: Removed redundant constants in TaskCard.js

- Untrimmed task titles: addTask stored titles without trimming even though the form validated .trim(). 
Fix: Titles are now trimmed before saving

- Stale category filter: Deleting the last task in a category removes it from the dropdown, but the filter still
holds the old value, returning 0 results. Fix: Filters reset when categories are removed

## What I learned
I learned that automated tests, including those generated by AI, are useful for catching many issues but are not enough on 
their own. Manual testing remains essential for identifying edge cases that automated tools can miss. I also found that 
providing clear and detailed prompts significantly improves the quality of AI-generated code. Finally, building features 
in small, testable phases made debugging more manageable and helped maintain a stable application throughout development.
