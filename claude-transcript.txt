  ---
  Phase 1 — Data Foundation (Build First)

  Before any UI, define your task shape and localStorage logic.

  useTasks.js custom hook

  This is the backbone of the whole app. It manages state and persistence.

  // What a task object looks like
  {
    id: "uuid-or-Date.now()",
    title: "Finish report",
    category: "Work",
    priority: "Top Priority",     // "Top Priority" | "Mid Priority" | "Low 
  Priority"
    dueDate: "2026-03-15",        // ISO string YYYY-MM-DD
    description: "",              // optional
    completed: false,
    createdAt: "2026-03-08T..."
  }

  localStorage structure

  // Key: "hype-pilot-tasks"
  // Value: JSON array of task objects
  localStorage.setItem("hype-pilot-tasks", JSON.stringify(tasks));

  React concepts used: useState, useEffect

  // useTasks.js pattern
  const [tasks, setTasks] = useState(() => {
    const saved = localStorage.getItem("hype-pilot-tasks");
    return saved ? JSON.parse(saved) : [];
  });

  useEffect(() => {
    localStorage.setItem("hype-pilot-tasks", JSON.stringify(tasks));
  }, [tasks]);

  The useState initializer function () => {...} runs only once — this is called
  lazy initialization and is the correct pattern for reading from localStorage.

  ---
  Phase 2 — Core Task CRUD

  Order to build:

  1. TaskForm.js — Controlled form with fields: title, category (dropdown or
  text), due date (<input type="date">), priority (select), description
  (textarea). Uses useState for each field. Calls an onAddTask(task) prop when
  submitted.
  2. TaskCard.js — Displays one task. Shows title, category badge, priority
  label, due date, and a checkbox to mark complete. Accepts task, onComplete,
  onDelete as props.
  3. TaskList.js — Maps over the tasks array and renders a <TaskCard> for each.
  Accepts tasks array as a prop.
  4. App.js — Pulls it all together using useTasks, renders <TaskForm> and
  <TaskList>.

  React concepts used: controlled components (value + onChange), props, lifting
  state up, .map() with key prop

  Phase 4 — Calendar View

  CalendarView.js — Groups tasks by due date and displays them in a weekly or
  monthly grid.

  Strategy: use a simple approach first — group tasks by date using reduce:

  const tasksByDate = tasks.reduce((acc, task) => {
    const date = task.dueDate;
    if (!acc[date]) acc[date] = [];
    acc[date].push(task);
    return acc;
  }, {});

  Then render a grid of the next 30 days, showing task counts or titles on their
   due date.

  ---
  Phase 5 — Reminders

  ReminderBanner.js — Displays a non-intrusive banner at the top listing tasks
  due within the next 3 days (and not yet completed).

  Filter logic:
  const dueSoon = tasks.filter(t =>
